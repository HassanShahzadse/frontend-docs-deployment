# UTXO Value Bands

The **UTXO Value Bands** metric distributes the active Bitcoin supply across UTXO value size cohorts, revealing how wealth concentrates across different output sizes. {{ className: 'lead' }}

This metric identifies the amount of Bitcoin held in outputs of various sizes, enabling analysis of whether supply concentrates in numerous small retail outputs or fewer large whale positions.

---

## Endpoint

`GET /utxo-value-bands`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

### Interpretation

**High small value band supply** indicates substantial Bitcoin locked in dust and micro outputs, revealing retail fragmentation where significant supply disperses across countless tiny uneconomical positions.

**High large value band supply** demonstrates massive Bitcoin concentration in whale-sized outputs, showing substantial supply consolidates in fewer but much larger positions held by sophisticated capital.

**Growing small value band supply** reveals increasing retail fragmentation, signaling Bitcoin supply is dispersing into smaller outputs through distribution or new participant influx creating numerous modest positions.

**Growing large value band supply** signals whale accumulation and consolidation, indicating sophisticated capital is concentrating supply into larger discrete positions through acquisition and output merging.

### Use Cases

This metric provides insight into how Bitcoin supply distributes across different output sizes, revealing wealth concentration versus dispersion patterns. High large value band supply indicates substantial capital consolidates in whale-sized outputs, suggesting sophisticated holders maintain significant positions in discrete large outputs rather than fragmenting across many addresses. These concentrated large positions represent patient capital with conviction, as maintaining substantial amounts in single outputs demonstrates confidence and long-term orientation. Conversely, high small value band supply reveals supply fragmentation across countless retail-sized outputs, indicating wealth disperses among many participants holding modest individual positions. The metric tracks wealth concentration shifts through supply migration between bands, with growing large bands confirming accumulation concentrates supply while growing small bands indicate distribution disperses holdings. Comparing supply amounts to UTXO counts in each band reveals average output sizes, with high supply relative to counts indicating fewer but larger outputs while low supply suggests many small outputs. This analysis identifies whether bands represent consolidated positions or fragmented holdings. Rising large value band supply during bear markets confirms accumulation as sophisticated capital consolidates supply into substantial positions. Declining large value band supply during bull markets warns of whale distribution as patient capital fragments large positions into smaller outputs for gradual selling. The metric also reveals retail participation through small band supply growth, with increasing amounts indicating new participants enter or existing holders accumulate into retail-sized positions.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/utxo-value-bands', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `utxo-value-bands?window=${window}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      range_0_001: d.range_0_001,
      range_001_01: d.range_001_01,
      range_01_1: d.range_01_1,
      range_1_10: d.range_1_10,
      range_10_100: d.range_10_100,
      range_100_1k: d.range_100_1k,
      range_1k_10k: d.range_1k_10k,
      range_10k_inf: d.range_10k_inf,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
