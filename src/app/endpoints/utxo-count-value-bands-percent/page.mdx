# UTXO Count - Value Bands (%)

The **UTXO Count - Value Bands (%)** metric distributes the total number of UTXOs across value size cohorts as percentages, revealing the relative fragmentation composition. {{ className: 'lead' }}

This metric identifies the proportional distribution of output counts by value tier, enabling normalized analysis of whether small retail dust or large whale outputs dominate the UTXO set structure.

---

## Endpoint

`GET /utxo-count-value-bands-percent`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

### Interpretation

**High small value percentages** indicate UTXO set dominated by dust and micro outputs, revealing retail fragmentation dominates the output count distribution despite representing minimal economic value, creating blockchain bloat.

**High large value percentages** demonstrate substantial proportion of outputs are whale-sized, showing large holders maintain fragmented positions that represent significant share of total UTXO count despite their small number.

**Rising small value percentages** reveal growing dust accumulation proportionally dominates UTXO set growth, signaling retail activity creates fragmented outputs faster than consolidation, increasing blockchain resource burden.

**Declining small value percentages** show dust cleanup or consolidation proportionally exceeds new small output creation, indicating effective UTXO management reduces the relative burden of uneconomical outputs.

### Use Cases

This metric provides normalized insight into UTXO set composition that enables meaningful comparison across different network states and total UTXO count levels. The percentage view removes the influence of absolute UTXO count changes, focusing on the relative character of fragmentation regardless of total output numbers. High small value percentages indicate the UTXO set is dominated by dust and micro outputs despite their minimal economic significance, revealing blockchain bloat from retail fragmentation. These conditions show the network carries substantial infrastructure burden from countless tiny uneconomical outputs that represent negligible value but consume storage and validation resources. The metric quantifies dust accumulation severity through small band percentage growth, with rising percentages indicating dust creation outpaces cleanup and increasingly burdens the network. Conversely, declining small value percentages signal successful dust management where consolidation or reduced retail activity decreases the relative proportion of uneconomical outputs. Comparing percentage distributions across cycles reveals whether current fragmentation patterns demonstrate similar or different characteristics than historical precedent. The metric also identifies retail versus whale fragmentation dominance, with heavily skewed distributions toward small values indicating retail-created dust dominates while more balanced distributions suggest diverse participant sizes maintain comparable output counts. Rising middle-band percentages during bull markets confirm retail graduation as small holders accumulate into larger positions, visible through growing mid-tier output proportions. The metric proves valuable for assessing blockchain efficiency, as high small value percentages indicate poor UTXO hygiene where the set is bloated with uneconomical outputs.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/utxo-count-value-bands-percent', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `utxo-count-value-bands-percent?window=${window}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      range_0_001_percent: d.range_0_001_percent,
      range_001_01_percent: d.range_001_01_percent,
      range_01_1_percent: d.range_01_1_percent,
      range_1_10_percent: d.range_1_10_percent,
      range_10_100_percent: d.range_10_100_percent,
      range_100_1k_percent: d.range_100_1k_percent,
      range_1k_10k_percent: d.range_1k_10k_percent,
      range_10k_inf_percent: d.range_10k_inf_percent,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
