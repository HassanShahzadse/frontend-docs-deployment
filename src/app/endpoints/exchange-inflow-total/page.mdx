# Exchange Inflow (Total)

The **Exchange Inflow (Total)** metric measures the total amount of Bitcoin transferred into exchange wallets during a specified time window. {{ className: 'lead' }}

Tracking total BTC inflows helps analysts identify when large volumes are being deposited to exchanges — often interpreted as increased potential for selling activity or heightened market volatility.

---

## Endpoint

`GET /exchange-inflow-total`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

---

## Example using filters

<Row>
  <Col>

Retrieve total BTC inflow data by exchange and timeframe. You can adjust parameters to specify the exchange, time window, and response format.

<Properties>
  <Property name="exchange" type="string">
    Exchange name. Example: <code>binance</code>, <code>coinbase</code>,{' '}
    <code>kraken</code>. Default: <code>all_exchange</code>.
  </Property>
  <Property name="window" type="string">
    Time resolution. Supported values: <code>day</code>, <code>hour</code>,{' '}
    <code>block</code>.
  </Property>
  <Property name="from" type="string">
    Start time in <code>YYYYMMDDTHHMMSS</code> or <code>YYYYMMDD</code>. For{' '}
    <code>window=block</code>, specify block height (e.g., <code>510000</code>).
  </Property>
  <Property name="to" type="string">
    End time in <code>YYYYMMDDTHHMMSS</code> or <code>YYYYMMDD</code>. For{' '}
    <code>window=block</code>, specify block height.
  </Property>
  <Property name="limit" type="integer">
    Maximum number of rows to return (1–100,000). Default: <code>100</code>.
  </Property>
  <Property name="format" type="string">
    Output format. Supported values: <code>json</code>, <code>csv</code>.
    Default: <code>json</code>.
  </Property>
</Properties>

  </Col>
  <Col>

```bash {title="Requesting Exchange Inflow (Total) data"}
curl -G "https://api.blocklytics.net/api/public/v1/exchange-inflow-total" \
  -H "x-api-key: YOUR_API_KEY" \
  -d exchange=binance \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
  -d limit=2
```


## Understanding the metric

<Row>
  <Col>

The **Exchange Inflow (Total)** metric provides valuable insight into exchange activity and investor sentiment:

<Properties>
  <Property name="inflow_total" type="number">
    Total BTC deposited to exchanges during the specified window.
  </Property>
  <Property name="inflow_top10" type="number">
    Total BTC deposited by the 10 largest inflow transactions.
  </Property>
  <Property name="inflow_mean" type="number">
    Average BTC inflow per transaction.
  </Property>
  <Property name="inflow_mean_ma7" type="number">
    7-day moving average of mean BTC inflow.
  </Property>
</Properties>

Sharp increases in **inflow_total** can indicate growing sell-side pressure or preparation for market events, while declining inflows may suggest investor accumulation or reduced liquidity movement.

  </Col>
  <Col>

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/exchange-inflow-total', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const exchange = req.query.exchange || 'all_exchange'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `exchange-inflow-total?window=${window}&exchange=${exchange}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      inflow_total: d.inflow_total,
    }))

    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
