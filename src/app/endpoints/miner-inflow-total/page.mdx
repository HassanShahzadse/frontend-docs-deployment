# Miner Inflow (Total)

The **Miner Inflow Total** metric tracks the total volume of BTC flowing into mining pool wallets. {{ className: 'lead' }}

This metric reveals overall miner revenue including mining rewards, external payments, and operational fund returns, providing a comprehensive view of total capital entering miner operations.

---

## Endpoint

`GET /miner-inflow-total`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="miner_inflow_total" type="number">
    The total volume of BTC flowing into mining pool wallets during the
    specified time window, including all sources of inflows.
  </Property>
</Properties>

### Interpretation

**Rising total inflow** indicates increasing capital entering miner operations, typically from mining rewards during high-fee periods, external fund returns from exchanges or storage, or operational capital injections.

**Declining total inflow** suggests reduced revenue capture by miners, which may result from lower fees, decreased mining activity, or periods when external fund sources contribute less to overall inflows.

**Total inflow significantly above expected mining rewards** reveals substantial non-mining capital entering miner wallets, such as returns from exchange sales, cold storage withdrawals, or other operational fund movements beyond standard block rewards.

**Total inflow closely matching expected rewards** indicates that mining rewards dominate inflows with minimal external capital movements, suggesting miners are primarily relying on mining revenue without significant treasury operations.

### Use Cases

This metric provides the foundation for understanding overall miner revenue and operational scale. Comparing total inflow with expected mining rewards reveals what percentage comes from actual mining versus external sources. Rising total inflows during bull markets often indicate miners benefiting from elevated transaction fees alongside price appreciation. When total inflow exceeds mining rewards substantially, it signals miners are bringing external capital into their operational wallets, possibly preparing for deployments or consolidating holdings. Tracking total inflow trends helps assess mining industry health, as sustained high inflows indicate robust revenue conditions while declining inflows may signal profitability pressures. For market analysis, comparing miner inflow with outflow provides net position data showing whether miners are accumulating or distributing their holdings.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/miner-inflow-total', async (req, res) => {
  try {
    const miner = req.query.miner || 'all_miner'
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `miner-inflow-total?window=${window}&miner=${miner}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      miner_inflow_total: d.miner_inflow_total,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
