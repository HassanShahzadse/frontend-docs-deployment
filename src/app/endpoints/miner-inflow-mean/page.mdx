# Miner Inflow (Mean)

The **Miner Inflow Mean** metric tracks the average transaction value flowing into mining pool wallets. {{ className: 'lead' }}

This metric reveals typical transaction sizes miners receive, helping distinguish between mining rewards, payment distributions, and other inflow sources based on transaction size patterns.

---

## Endpoint

`GET /miner-inflow-mean`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="miner_inflow_mean" type="number">
    The average value of transactions flowing into mining pool wallets during
    the specified time window, denominated in BTC.
  </Property>
</Properties>

### Interpretation

**Rising mean inflow** indicates larger average transactions entering miner wallets, which may reflect block reward increases from fee spikes, larger external payments, or changes in payout structures favoring bigger batch transfers.

**Declining mean inflow** suggests smaller average transactions, potentially indicating more distributed payment patterns, lower fees reducing overall reward sizes, or shifts toward more frequent smaller payouts.

**Consistent mean values around block reward size** confirm that inflows primarily consist of mining rewards, with typical values reflecting the current block subsidy plus average transaction fees earned.

**Mean values significantly different from block rewards** reveal substantial non-mining inflows, such as purchases from exchanges, returns from cold storage, or other operational fund movements beyond standard mining revenue.

### Use Cases

This metric helps distinguish between different types of miner inflows and understand revenue composition. Mean inflow values close to block reward amounts suggest mining rewards dominate, while significantly lower means indicate many small transactions from other sources. Tracking mean inflow changes helps identify when miners adjust payout policies or when external fund sources become more significant. Comparing mean inflow across mining pools reveals different operational structures, with some maintaining simpler direct reward collection and others showing more complex fund flows. Rising mean inflows during high-fee periods confirm miners are capturing elevated transaction fees effectively. For analyzing miner economics, sustained high mean inflows relative to historical averages indicate healthy revenue conditions, while declining means may signal profitability pressures.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/miner-inflow-mean', async (req, res) => {
  try {
    const miner = req.query.miner || 'all_miner'
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `miner-inflow-mean?window=${window}&miner=${miner}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      miner_inflow_mean: d.miner_inflow_mean,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
