# Fees (Total)

The **Fees Total** metric tracks the sum of all transaction fees paid to miners during a specified time window. {{ className: 'lead' }}

This metric reveals network usage intensity and demand for block space. Rising fees indicate increased transaction activity and competition for confirmation, while declining fees suggest lower network congestion.

---

## Endpoint

`GET /fees-total`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

## Understanding the metric

  <Col>

### Interpretation

**Rising total fees** indicate increased network activity and higher demand for block space, often signaling periods of intense trading, NFT minting waves, or major market movements that drive transaction volume.

**Declining total fees** suggest reduced network congestion and lower transaction activity, which may indicate quieter market periods, decreased user engagement, or improved layer-2 adoption reducing on-chain pressure.

**Fee spikes** typically occur during major market events, popular token launches, network congestion from high-value transactions, or periods when users compete aggressively for fast confirmation times.

**Sustained high fees** can indicate structural network demand growth, increased adoption driving consistent activity, or persistent congestion requiring scaling solutions.

### Use Cases

This metric is essential for understanding network health and miner economics. Total fees represent an increasingly important component of miner revenue, especially as block subsidies decrease over time through halvings. High fee periods often correlate with major price movements or significant on-chain events. Tracking fee trends helps predict miner behavior, as miners become more dependent on transaction fees for profitability. When combined with block reward data, you can calculate the percentage of miner revenue coming from fees versus block subsidies, revealing the network's progress toward a fee-based security model.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/fees-total', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `fees-total?window=${window}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      fees_total: d.fees_total,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
