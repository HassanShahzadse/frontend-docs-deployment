# Miner Depositing Addresses

The **Miner Depositing Addresses** metric tracks the number of unique addresses involved in inflow transactions to mining pools. {{ className: 'lead' }}

This metric reveals the diversity and distribution of inflows to miners, helping identify changes in miner revenue sources and transaction patterns.

---

## Endpoint

`GET /miner-depositing-addresses`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

```bash {title="Requesting Miner Depositing Addresses data"}
curl -G "https://api.blocklytics.net/api/public/v1/miner-depositing-addresses" \
  -H "x-api-key: YOUR_API_KEY" \
  -d miner=f2pool \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
  -d limit=2
```

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="miner_depositing_addresses" type="number">
    The number of unique addresses sending transactions to the mining pool
    during the specified time window.
  </Property>
</Properties>

### Interpretation

**Rising depositing addresses** indicate increasing interaction with the mining pool, which may reflect growing miner participation, more distributed hashrate, or increased transaction activity involving miner wallets.

**Declining depositing addresses** suggest reduced interaction with the mining pool, potentially indicating consolidation among fewer participants or decreased transaction activity to miner addresses.

**High address count relative to transaction volume** reveals distributed activity with many smaller participants interacting with the mining pool, suggesting a diverse ecosystem of individual miners or smaller operations.

**Low address count relative to transaction volume** indicates concentrated activity where fewer addresses send larger amounts, possibly reflecting institutional mining operations or larger consolidation among participants.

### Use Cases

This metric helps assess the distribution and diversity of miner ecosystem participation. Rising address counts suggest growing participation in mining operations, indicating healthy distribution and potentially increasing decentralization. Comparing address counts across different mining pools reveals which operations attract more diverse participation versus concentrated institutional players. Sudden changes in depositing addresses can signal shifts in mining pool popularity, changes in payout policies, or migration of hashrate between pools. For analyzing miner behavior, combining this metric with inflow volume data reveals whether activity is driven by many small participants or few large ones, providing insights into mining industry structure and concentration trends.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/miner-depositing-addresses', async (req, res) => {
  try {
    const miner = req.query.miner || 'all_miner'
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `miner-depositing-addresses?window=${window}&miner=${miner}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      miner_depositing_addresses: d.miner_depositing_addresses,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
