# Exchange to Exchange Transactions

The **Exchange to Exchange Transactions** metric tracks the number of transactions occurring between exchanges. {{ className: 'lead' }}

This metric reveals the frequency of inter-exchange transfers, helping identify active arbitrage corridors, automated trading bot activity, and the distribution of transfer sizes between platforms.

---

## Endpoint

`GET /exchange-to-exchange-transactions`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

```bash {title="Requesting Exchange to Exchange Transactions data"}
curl -G "https://api.blocklytics.net/api/public/v1/exchange-to-exchange-transactions" \
  -H "x-api-key: YOUR_API_KEY" \
  -d from_exchange=binance \
  -d to_exchange=coinbase \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
  -d limit=2
```

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="transactions_count_flow" type="integer">
    The number of transactions flowing from the source exchange to the
    destination exchange during the specified time window.
  </Property>
</Properties>

### Interpretation

**Rising transaction count** indicates increased inter-exchange activity, often signaling active arbitrage opportunities, automated bot operations, or heightened price discrepancies between platforms driving frequent transfers.

**Declining transaction count** suggests reduced inter-exchange movement, which may indicate aligned pricing across platforms, reduced arbitrage opportunities, or decreased automated trading activity.

**High count with low total volume** (from Flow Total metric) reveals many small transactions, typically indicating retail arbitrage activity or distributed automated trading strategies across multiple smaller positions.

**Low count with high total volume** suggests fewer but larger institutional transfers, indicating whale movements or major rebalancing operations between platforms.

### Use Cases

This metric is crucial for distinguishing between retail and institutional inter-exchange activity. When combined with Flow Total and Flow Mean metrics, you can identify whether arbitrage is dominated by many small traders or few large players. Sudden increases in transaction count often precede or follow significant price movements on either platform. Monitoring hourly transaction patterns can reveal optimal arbitrage windows and identify automated bot activity schedules.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/exchange-to-exchange-transactions', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const fromExchange = req.query.from_exchange || 'all_exchange'
    const toExchange = req.query.to_exchange || 'spot_exchange'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `exchange-to-exchange-transactions?window=${window}&from_exchange=${fromExchange}&to_exchange=${toExchange}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      transactions_count_flow: d.transactions_count_flow,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
