# Exchange to Miner Transactions

The **Exchange to Miner Transactions** metric tracks the number of transactions occurring from exchanges to mining pools. {{ className: 'lead' }}

This metric reveals the frequency of miner purchasing activity and helps distinguish between strategic accumulation patterns and operational purchasing behavior.

---

## Endpoint

`GET /exchange-to-miner-transactions`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

```bash {title="Requesting Exchange to Miner Transactions data"}
curl -G "https://api.blocklytics.net/api/public/v1/exchange-to-miner-transactions" \
  -H "x-api-key: YOUR_API_KEY" \
  -d from_exchange=binance \
  -d to_miner=f2pool \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
  -d limit=2
```

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="transactions_count_flow" type="integer">
    The number of transactions flowing from the exchange to the mining pool
    during the specified time window.
  </Property>
</Properties>

### Interpretation

**Rising transaction count** indicates increased frequency of miner purchases from exchanges, suggesting active accumulation behavior or regular operational purchasing patterns. Higher frequency may signal sustained buying interest from mining operations.

**Declining transaction count** suggests reduced miner purchasing frequency, which may indicate miners are satisfied with current holdings, focusing on selling mined tokens, or awaiting more favorable price levels.

**High count with low total volume** (from Flow Total metric) reveals many small transactions, typically indicating operational purchasing for expenses, fees, or distributed buying strategies to minimize market impact.

**Low count with high total volume** suggests fewer but larger strategic purchases, indicating concentrated accumulation events by well-capitalized mining operations making significant position entries.

### Use Cases

This metric is crucial for understanding miner purchasing behavior and distinguishing between operational and strategic activity. When combined with Flow Total and Flow Mean metrics, you can identify whether miners are making regular small purchases for operations or executing large strategic accumulation. Sudden increases in transaction count during price dips often signal miner confidence in current valuations. Monitoring specific mining pool transaction patterns reveals which operations are most active in market participation and which maintain more passive holding strategies. Regular transaction patterns can indicate scheduled purchasing programs by mining operations with strong treasury management.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/exchange-to-miner-transactions', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const fromExchange = req.query.from_exchange || 'all_exchange'
    const toMiner = req.query.to_miner || 'all_miner'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `exchange-to-miner-transactions?window=${window}&from_exchange=${fromExchange}&to_miner=${toMiner}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      transactions_count_flow: d.transactions_count_flow,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
