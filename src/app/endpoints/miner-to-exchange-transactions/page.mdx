# Miner to Exchange Transactions

The **Miner to Exchange Transactions** metric tracks the total number of transactions transferring Bitcoin from mining pool wallets to exchange platforms. {{ className: 'lead' }}

This metric reveals the frequency and operational structure of miner-to-exchange flows, distinguishing between consolidated bulk transfers and fragmented distribution patterns.

---

## Endpoint

`GET /miner-to-exchange-transactions`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

```bash {title="Requesting Miner to Exchange Transactions data"}
curl -G "https://api.blocklytics.net/api/public/v1/miner-to-exchange-transactions" \
  -H "x-api-key: YOUR_API_KEY" \
  -d from_miner=f2pool \
  -d to_exchange=binance \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
```

## Understanding the metric

<Row>
  <Col>

<Properties>
  <Property name="transactions_count_flow" type="number">
    The total number of transactions transferring Bitcoin from the specified
    mining pool to the specified exchange during the time period.
  </Property>
</Properties>

### Interpretation

**Rising transaction counts** indicate increased frequency of miner-to-exchange transfers, suggesting more active selling operations, fragmented distribution strategies, or operational changes favoring multiple smaller deposits over consolidated batches.

**Declining transaction counts** suggest miners are consolidating exchange transfers into fewer transactions, potentially indicating operational efficiency improvements, reduced selling frequency, or strategic shifts toward bulk deposits that minimize transaction overhead.

**High transaction counts with moderate volumes** reveal fragmented distribution patterns where miners spread exchange deposits across many smaller transactions, often indicating incremental selling strategies, automated regular deposits, or efforts to minimize market impact through dispersed timing.

**Low transaction counts with high volumes** indicate concentrated transfer operations where miners execute large deposits in few transactions, typically suggesting institutional-grade efficiency, direct exchange relationships with bulk transfer arrangements, or strategic timing of major liquidations.

### Use Cases

This metric provides crucial insight into the operational sophistication and selling strategies of mining operations in their exchange interactions. Transaction frequency patterns reveal how miners structure their exchange deposits, with high-frequency patterns indicating active continuous selling while low-frequency patterns suggest periodic consolidated operations. Comparing transaction counts to total flow volumes calculates average transaction sizes, revealing whether miners favor many small deposits or fewer large transfers. During bull markets, rising transaction counts often signal miners actively capitalizing on favorable conditions through increased exchange deposit frequency, while declining counts during bear markets may indicate reduced selling activity or strategic consolidation. Tracking transaction counts alongside mean flow sizes distinguishes between operational efficiency changes and volume shifts, where declining counts with stable means indicate consolidation while rising counts with stable means reveal expanded operations. This metric helps identify strategic shifts in miner behavior, such as transitions between automated regular selling programs with high transaction frequencies and manual strategic selling with concentrated transactions. Analyzing transaction count patterns by specific exchange destinations reveals preferred operational relationships and liquidity management approaches, with consistent high-frequency deposits to particular platforms often indicating established automated distribution systems. Comparing transaction counts across different mining pools reveals operational differences in treasury management, with some operations demonstrating sophisticated batch processing while others maintain frequent incremental approaches.

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/miner-to-exchange-transactions', async (req, res) => {
  try {
    const fromMiner = req.query.from_miner || 'all_miner'
    const toExchange = req.query.to_exchange || 'all_exchange'
    const window = req.query.window || 'day'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `miner-to-exchange-transactions?window=${window}&from_miner=${fromMiner}&to_exchange=${toExchange}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      transactions_count_flow: d.transactions_count_flow,
    }))
    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
