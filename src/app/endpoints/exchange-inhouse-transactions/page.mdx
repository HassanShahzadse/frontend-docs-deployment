# Exchange In-House Transactions

The **Exchange In-House Transactions** metric tracks the total number of Bitcoin transactions between wallets belonging to the same exchange. {{ className: 'lead' }}

This indicator helps monitor the level of internal fund reorganization, custodial maintenance, and liquidity balancing among exchange-operated wallets.

---

## Endpoint

`GET /exchange-inhouse-transactions`

**Base URL:**  
`https://api.blocklytics.net/api/public/v1`

---

This example retrieves the number of internal transactions per day. Adjust parameters to query a specific exchange, timeframe, or export format.

  <Property name="to" type="string">
    End time in <code>YYYYMMDDTHHMMSS</code> or <code>YYYYMMDD</code>. For{' '}

  </Property>

  </Property>

  </Col>

```bash {title="Requesting Exchange In-House Transactions data"}
curl -G "https://api.blocklytics.net/api/public/v1/exchange-inhouse-transactions" \
  -H "x-api-key: YOUR_API_KEY" \
  -d exchange=binance \
  -d window=day \
  -d from=20250101 \
  -d to=20250107 \
  -d limit=2
```

```json {title="Sample JSON response"}

    "date": "2025-01-07",

```

```bash {title="Exporting as CSV"}

  -H "x-api-key: YOUR_API_KEY" \
  -d exchange=kraken \
  -d window=hour \
  -d from=2025010100 \
  -d to=2025010123 \
  -d format=csv
```

## Understanding the metric

<Row>
  <Col>

The **Exchange In-House Transactions** metric reveals operational activity levels within exchanges:

<Properties>
  <Property name="transactions_count_flow" type="integer">
    Number of Bitcoin transactions occurring internally within exchange wallets.
  </Property>
  <Property name="flow_mean" type="number">
    Average transaction size of these internal transfers.
  </Property>
  <Property name="flow_total" type="number">
    Total BTC transferred internally between exchange wallets.
  </Property>
  <Property name="exchange" type="string">
    Name of the exchange for which the internal flow data is tracked.
  </Property>
</Properties>

High internal transaction counts may indicate large-scale operational adjustments, hot/cold wallet rebalancing, or custodial fund movements. Lower counts usually suggest stable operations.

  </Col>
  <Col>

```js {title="Node.js (Express) example"}
// Example Express route using Blocklytics
router.get('/exchange-inhouse-transactions', async (req, res) => {
  try {
    const window = req.query.window || 'day'
    const exchange = req.query.exchange || 'all_exchange'
    const from = req.query.from ? `&from=${req.query.from}` : ''
    const to = req.query.to ? `&to=${req.query.to}` : ''
    const limit = req.query.limit ? `&limit=${req.query.limit}` : ''
    const format = req.query.format ? `&format=${req.query.format}` : ''

    const path = `exchange-inhouse-transactions?window=${window}&exchange=${exchange}${from}${to}${limit}${format}`
    const data = await blocklyticsFetch(path)

    if (!Array.isArray(data)) {
      return res.status(500).json({ error: 'Unexpected API format', data })
    }

    const formatted = data.map((d) => ({
      date: d.date,
      transactions_count_flow: d.transactions_count_flow,
    }))

    res.json(formatted)
  } catch (err) {
    res.status(500).json({ error: err.message })
  }
})
```
